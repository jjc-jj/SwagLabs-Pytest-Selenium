##### 🔧 国内网络环境专项下载指南

**1. Java JDK（JMeter 前置依赖）**

```
方案 A（推荐）：清华镜像站下载 OpenJDK
  - 地址：https://mirrors.tuna.tsinghua.edu.cn/Adoptium/
  - 选择：Windows x64 → JDK 11 → .msi 安装包
  - 安装后验证：打开终端输入 java -version，显示版本号即成功

方案 B：华为镜像站
  - 地址：https://mirrors.huaweicloud.com/openjdk/
  - 同样选 JDK 11 Windows 版本

⚠️ 不要装 JDK 17 或更高版本——JMeter 5.x 启动可能报兼容性错误
```

**2. JMeter（国内镜像加速）**

```
方案 A（推荐）：阿里云 Apache 镜像
  - 地址：https://mirrors.aliyun.com/apache/jmeter/binaries/
  - 下载：apache-jmeter-5.6.3.zip（约 65MB）
  - 解压到 D:\tools\apache-jmeter-5.6.3\
  - 启动：双击 bin\jmeter.bat

方案 B：清华 Apache 镜像
  - 地址：https://mirrors.tuna.tsinghua.edu.cn/apache/jmeter/binaries/
  - 同上操作

⚠️ 解压路径不要有中文和空格（如 D:\软件\Jmeter ❌ → D:\tools\jmeter ✅）
```

**3. Postman（替代方案）**

```
方案 A（推荐）：使用 Apifox 或 Apipost（国产替代，功能相近，国内下载快）
  - Apifox：https://apifox.com/  （接口测试 + Mock + 文档，一个工具全搞定）
  - Apipost：https://www.apipost.cn/
  - 面试时完全可以说"我用的 Apifox 做接口测试"，不影响评价

方案 B：如果坚持用 Postman
  - 官网下载可能极慢（150MB 境外服务器）
  - 尝试从"腾讯软件中心"或"联想应用商店"搜索 Postman 国内加速版
  - 或者让朋友从网盘传一个安装包
```

**4. ChromeDriver（最易翻车的一项）**

```
问题：ChromeDriver 托管在 Google 服务器（storage.googleapis.com），十堰可能直接下载失败。

方案 A（推荐）：淘宝 npm 镜像
  - 地址：https://registry.npmmirror.com/binary.html?path=chromedriver/
  - 选择与你的 Chrome 版本号匹配的版本（Chrome 版本查看：地址栏输入 chrome://version）
  - 下载 chromedriver_win32.zip

方案 B：自动匹配（用 Python 库）
  pip install webdriver-manager -i https://pypi.tuna.tsinghua.edu.cn/simple
  # 代码中这样写，自动下载匹配的 ChromeDriver：
  from selenium import webdriver
  from webdriver_manager.chrome import ChromeDriverManager
  from selenium.webdriver.chrome.service import Service
  driver = webdriver.Chrome(service=Service(ChromeDriverManager().install()))

  ⚠️ webdriver-manager 首次运行仍需访问 Google CDN，建议方案 A + B 同时准备

方案 C（终极兜底）：使用 Edge 浏览器 + EdgeDriver
  - Edge 是 Chromium 内核，Selenium 完全兼容
  - EdgeDriver 由微软托管，国内下载无障碍
  - 地址：https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/
  - 代码中把 Chrome() 换成 Edge() 即可
```

#### 关键代码片段示例

```python
# test_login.py
import pytest
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

class TestLogin:
    """登录功能自动化测试"""
    
    BASE_URL = "https://www.saucedemo.com/"
    
    @pytest.fixture(autouse=True)
    def setup(self):
        self.driver = webdriver.Chrome()
        self.driver.get(self.BASE_URL)
        yield
        self.driver.quit()
    
    def test_login_success(self):
        """TC-AUTO-001：合法用户登录成功"""
        self.driver.find_element(By.ID, "user-name").send_keys("standard_user")
        self.driver.find_element(By.ID, "password").send_keys("secret_sauce")
        self.driver.find_element(By.ID, "login-button").click()
        
        # 显式等待 + 断言
        WebDriverWait(self.driver, 10).until(
            EC.presence_of_element_located((By.CLASS_NAME, "inventory_list"))
        )
        assert "inventory.html" in self.driver.current_url
    
    @pytest.mark.parametrize("username,password,expected_error", [
        ("standard_user", "wrong_password", "do not match"),
```

##### 任务 A：Mock 异常注入（接口层）

在 Pytest 自动化框架中，使用 **responses** 或 **unittest.mock** 库拦截 HTTP 请求，模拟极端异常场景，验证前端的容错处理：

| Mock 场景 | 模拟内容 | 验证点 |
|-----------|---------|--------|
| 库存不足 | Mock 加购接口返回 `{"error": "out_of_stock"}` + 500 状态码 | 前端是否展示友好错误提示，而非空白页或崩溃 |
| 网络超时 | Mock 请求延迟 10s 后抛出 `Timeout` | 是否有超时提示或 loading 状态，脚本是否正确捕获超时异常 |
| Token 过期 | Mock 接口返回 401 Unauthorized | 是否自动跳转登录页，而非停留在当前页显示异常数据 |
| 数据为空 | Mock 商品列表返回 `[]` | 是否展示"暂无商品"空状态页，而非报错 |

**代码示例（Pytest + responses 库）**：

```python
import responses
import requests

class TestMockScenarios:
    """Mock 异常注入测试"""

    @responses.activate
    def test_out_of_stock_shows_friendly_error(self, driver):
        """MOCK-SC-001：Mock 库存不足，验证前端容错"""
        # Mock 加购接口返回 500
        responses.add(
            responses.POST,
            "https://www.saucedemo.com/api/cart/add",
            json={"error": "out_of_stock", "message": "该商品库存不足"},
            status=500
        )
        # ... 执行加购操作 ...
        # 断言：前端展示友好提示，而非崩溃
        error_msg = driver.find_element(By.CSS_SELECTOR, ".error-message")
        assert "库存不足" in error_msg.text or "error" in error_msg.text.lower()

    @responses.activate
    def test_token_expiry_redirects_to_login(self, driver):
        """MOCK-SC-002：Mock Token 过期，验证自动跳转登录页"""
        responses.add(
            responses.GET,
            "https://www.saucedemo.com/api/inventory",
            json={"error": "unauthorized"},
            status=401
        )
        # 刷新商品页 → 应跳转登录
        driver.refresh()
        WebDriverWait(driver, 5).until(
            EC.presence_of_element_located((By.ID, "login-button"))
        )
        assert "saucedemo.com" in driver.current_url
        # 确认 URL 中不含 inventory（已从商品页跳离）
        assert "inventory" not in driver.current_url
```

### 第六阶段：JMeter 性能测试 — 跳过

> **⚠️ 技术评估结论**：性能测试需要同时满足以下前提条件——可控的服务器环境、服务端监控（CPU/内存/DB）、可重复的测试环境、可对比的基线数据、稳定的网络链路。精品零售Web系统（saucedemo.com）作为公开演示平台，**以上条件全部不满足**。

#### 跳过不等于放弃

性能测试的核心能力——识别瓶颈、分析 TPS/响应时间/错误率、定位优化方向——需要在**可控制、可监控、可优化**的真实项目环境中才能真正掌握。在 精品零售Web系统 上强行执行，只会学到"如何在 JMeter 里配一个 HTTP 请求然后看几个数字"，这不是性能测试。

### 风险 2：精品零售Web系统 的反爬/限流机制 → 脚本间歇性失败

**问题诊断**：精品零售Web系统 是公开平台，如果自动化脚本运行频率过高（比如短时间内连续登录几十次），可能触发验证码或 IP 临时封锁。这会导致脚本报错 `NoSuchElementException` 或 `TimeoutException`——**但这不是你的代码问题**，是平台在限流。

**纠偏措施**：

**A. 代码层：用随机等待替代固定等待**

```python
# ❌ 不要这样写
time.sleep(2)

# ✅ 应该这样写
import random
time.sleep(random.uniform(1, 3))  # 1~3 秒随机，模拟人类操作节奏
```

在 `base_page.py` 中封装一个 `human_delay()` 方法，所有页面操作后调用：

```python
class BasePage:
    @staticmethod
    def human_delay(min_s=0.5, max_s=2.0):
        """模拟人类操作间隔，避免触发反爬"""
        import random, time
        time.sleep(random.uniform(min_s, max_s))
```

### 风险 3：微创新的"含金量"不足 → 面试官觉得在凑字数

**问题诊断**：方案中提到了"画架构图"和"Mock 测试"，但如果只是浅尝辄止——比如画了个简陋的方框图、Mock 了一个简单的 200 响应——面试官一眼就能看出是在凑字数，反而扣分。

**纠偏措施：做到"狠"和"脏"**

**A. 架构图要"脏"——越满越真实**

不要画干净的示意图。把以下信息全部堆上去：

| 元素 | 要求 |
|------|------|
| **真实接口路径** | 如 `POST /api/v1/cart/add` → 不要自己编，用 Fiddler 抓包拿真实路径 |
| **请求参数名** | 如 `item_id`, `quantity`, `session_token` — 截图 JSON 字段名贴上去 |
| **返回 JSON 片段** | 如 `{"cart_id": 123, "item_count": 3}` — 截图贴旁边 |
| **异常分支** | 在图中用红色箭头标出：Token 过期 → 401、库存不足 → 500 |
| **HTTP 状态码** | 在每个接口旁标注 200/302/401/500 |

工具推荐：**Draw.io（桌面版）** 或 **ProcessOn（在线）**。导出的 PNG 放 GitHub 仓库 `docs/` 目录。

**B. Mock 要"狠"——不只 Mock 成功/失败**

| 级别 | Mock 内容 | 面试含金量 |
|------|----------|-----------|
| 入门 | Mock 500 错误 | ⭐⭐（所有人都做） |
| 进阶 | Mock 网络超时（`Timeout`） | ⭐⭐⭐ |
| **高级** | **Mock 弱网环境：用 Charles 的 Throttle 功能模拟 3G 网络（200KB/s，延迟 300ms）** | ⭐⭐⭐⭐⭐ |
| 高级 | Mock 接口返回空数组 `[]` → 验证空状态 UI | ⭐⭐⭐⭐ |
| 高级 | Mock 接口返回乱码/超长字段 → 验证前端截断防护 | ⭐⭐⭐⭐ |

> **面试金句**："我没有只 Mock 成功和失败，我用 Charles 的 Throttle 功能模拟了弱网环境——在 3G 网络条件下跑了一遍核心流程，发现页面加载超过 10 秒时没有骨架屏或 loading 动画，用户体验很差。我把这个问题写进了测试报告的性能建议部分。"

## 六、常见问题预案

### Q1：如果时间不够怎么办？

**降级方案**：
- 用例数量从 80 条降到 50 条（覆盖登录+购物车+结账三个核心模块）
- Selenium 脚本从 5 个场景降到 3 个（登录成功、加购物车、完整结账）
- JMeter 只做一个场景（登录接口）
- **底线**：手工用例 + Selenium 脚本 + Bug 报告这三样必须有

### Q2：精品零售Web系统 平台访问不了/触发反爬怎么办？

**原因判断**：
- 页面加载极慢（>10s）→ 网络问题，尤其十堰访问国外服务器不稳定
- 返回验证码/空白页 → 触发了反爬机制
- 页面结构变了 → 平台更新了前端代码

**应对方案**：

```
第一步（不中断工作）：立刻切换到本地备用靶场
  - Django Polls：部署只需 30 分钟，纯本地、零延迟、完全可控
  - 命令：django-admin startproject mysite && python manage.py startapp polls
  - 参考：https://docs.djangoproject.com/en/stable/intro/tutorial01/

第二步（排查 精品零售Web系统 问题）：
  - 反爬：降低脚本频率，加入 random.uniform(1, 3) 随机等待
  - 网络：挂代理或切换网络再试
  - 页面变更：用 DevTools 重新定位元素，更新 XPath/CSS Selector

第三步（面试话术准备）：
  - "为了保证测试的稳定性和可重复性，我除了在 精品零售Web系统 上做探索性测试外，
     还本地部署了一套 Django 应用，将自动化回归脚本跑在本地环境中，
     验证了同一套脚本在不同环境下的可移植性。"
```

### Q3：Python/Selenium 遇到技术问题怎么办？

- B 站搜"Selenium 自动化测试实战"（黑马/尚硅谷的免费教程）
- Selenium 官方文档：https://www.selenium.dev/documentation/
- 代码问题用 AI 工具辅助调试
