🔍 抓包验证步骤（面试时展示用）

# Step 1: 打开 DevTools F12 → Network 标签 → 勾选 "Preserve log" # Step 2: 完整走一遍流程 登录(standard_user) → 浏览商品 → 查看详情 → 加购 → 结账 → Finish # Step 3: 筛选关键请求 Network 面板搜索框输入: .html → 查看所有页面请求 Network 面板搜索框输入: inventory → 查看商品相关 # Step 4: 验证 Cookie 机制 点击任一请求 → Cookies 标签 → 确认 session-username 存在 清除 Cookie → 刷新 → 确认被重定向到登录页 # Step 5: 导出 HAR 文件（面试证据） Network 面板 → 右键 → Save all as HAR with content # 保存到 docs/network_trace.har，放入 GitHub 仓库
🔬 微创新要点（面试时主动展示）：
1. 架构图不是画出来的，是"抓"出来的——每个接口路径、Cookie 名称、页面 URL 都来自 DevTools 实际抓包，不是凭想象画
2. 异常路径不是臆想的——清除 Cookie 后重试、未登录直接访问 URL、输入空值提交，每个异常场景都有实际验证
3. 技术发现：Swag Labs 的"认证"仅依赖一个 Cookie（session-username），无 Token 过期机制、无 HttpOnly 标记——这是一个安全缺陷，已记录为 Bug
4. HAR 文件已导出保存至 docs/network_trace.har，面试时可现场导入 DevTools 回放完整请求链路
